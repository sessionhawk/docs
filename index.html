Complete Setup & Deployment Guide

This guide covers everything you need to deploy, update, and integrate SessionHawk for real-world session hijacking protection.

1. What is SessionHawk?

SessionHawk is a self-hosted session security service designed to protect your users from advanced threats. It operates by validating user sessions against a continuously evolving device profile.

Key Features

  • Detects session hijacking attempts in real-time.
  • Identifies stolen authentication cookies before they can be exploited.
  • Tracks device/browser fingerprint changes with high precision.
  • Flags suspicious network shifts (VPN, Tor, Proxy detection).
Data Storage: All data (fingerprints, users, groups, API keys) is stored inside the container at /app/data. You MUST mount this to a persistent volume on your host machine.

2. Initial Docker Deployment

Step 1: Pull the Latest Image

docker pull sessionhawk/sessionhawk:latest

Step 2: Run the Container

Run the container in detached mode with persistent storage enabled.

docker run -d \
  --name sessionhawk_container \
  -p 8080:8080 \
  -v $PWD/data:/app/data \
  sessionhawk/sessionhawk:latest

Command Explanation:

Flag Description
-d Run in background (detached mode).
--name Name your container "sessionhawk_container".
-p 8080:8080 Map port 8080 (host) to 8080 (container).
-v $PWD/data:/app/data Mount local ./data directory to persist data.

Step 3: View API Keys

View the logs to retrieve your auto-generated API keys.

docker logs -f sessionhawk_container

Look for output like: ‚úì API Key 1: sk_abc123.... Save these securely.

Step 4: Access the Service

Your instance is running at http://your-server-ip:8080. Check the health:

curl http://localhost:8080/health
# Expected response: 200 OK

3. Understanding Data Persistence

üîí CRITICAL: Your Data Location
When you run the container with -v $PWD/data:/app/data, all SessionHawk data is stored on your HOST at ./data/.

This directory contains:

  • All user fingerprints
  • All groups and users
  • All API keys
  • Session history & configuration

üö® Important Rules

  1. NEVER delete the ./data/ directory on your host.
  2. ALWAYS use the same volume mount when updating.
  3. Removing a container does NOT delete the volume.
  4. Your data survives container updates, stops, and removals.

4. How to Update Without Losing Data

When a new version is released, follow these steps EXACTLY to ensure zero data loss.

The Safe Update Procedure

  1. Stop the container: docker stop sessionhawk_container
  2. Remove the shell: docker rm sessionhawk_container (Your data in ./data/ remains safe).
  3. Pull latest image: docker pull sessionhawk/sessionhawk:latest
  4. Start new container: (Ensure you use the same volume mount)
docker run -d \
  --name sessionhawk_container \
  -p 8080:8080 \
  -v $PWD/data:/app/data \
  sessionhawk/sessionhawk:latest

üîÑ One-Command Update (Copy-Paste Ready)

docker stop sessionhawk_container && \
docker rm sessionhawk_container && \
docker pull sessionhawk/sessionhawk:latest && \
docker run -d \
  --name sessionhawk_container \
  -p 8080:8080 \
  -v $PWD/data:/app/data \
  sessionhawk/sessionhawk:latest

6. Complete API Reference

Authentication Headers Required (except /health):
X-API-Key: <your-api-key>
Content-Type: application/json

üìã All Available Endpoints

1. Health Check

GET /health — Does not require authentication

curl http://localhost:8080/health
# Response: 200 OK

2. API Key Management

DELETE /keys/{key} — Deletes an API key (cannot delete the last one)

curl -X DELETE http://localhost:8080/keys/sk_old_key_to_delete \
  -H "X-API-Key: YOUR_ADMIN_API_KEY"

3. Group Management

POST /groups — Create a new group

Use cases: Organize users by team, environment, or application.

curl -X POST http://localhost:8080/groups \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"group":"engineering"}'

DELETE /groups/{group} — Delete a group and all its users

curl -X DELETE http://localhost:8080/groups/engineering \
  -H "X-API-Key: YOUR_API_KEY"

4. User Management

POST /groups/{group}/users — Add user to group

curl -X POST http://localhost:8080/groups/engineering/users \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"user":"alice"}'

DELETE /groups/{group}/users/{user} — Remove user from group

curl -X DELETE http://localhost:8080/groups/engineering/users/alice \
  -H "X-API-Key: YOUR_API_KEY"

5. Fingerprint Storage

POST /groups/{group}/users/{user}/fp — Store a device fingerprint

Call this at user login or when registering a new device.

curl -X POST http://localhost:8080/groups/engineering/users/alice/fp \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "fingerprint": {
      "metadata": "combined",
      "browser_family": "Chrome",
      "os_family": "Windows",
      "canvas_pixel_hash": "abc123",
      "canvas_text_hash": "def456",
      "webgl_render_hash": "ghi789",
      "webgl_vendor_generalized": "Intel"
    }
  }'

DELETE /groups/{group}/users/{user}/fp/{index} — Delete fingerprint by index (0-based)

curl -X DELETE http://localhost:8080/groups/engineering/users/alice/fp/0 \
  -H "X-API-Key: YOUR_API_KEY"

6. Session Validation (Compare)

POST /compare — Validate current session against stored fingerprints

Call this on every page load or before sensitive actions.

curl -X POST http://localhost:8080/compare \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "group": "engineering",
    "user": "alice",
    "fingerprint": {
      "metadata": "combined",

7. View All Data

GET /dataset — Retrieve entire dataset (groups, users, fingerprints)

curl http://localhost:8080/dataset \
  -H "X-API-Key: YOUR_API_KEY"

üìä Compare Response Verdicts

Verdict Meaning Action
true Fingerprint matches at least one stored fingerprint Allow session to continue
false No matching fingerprint found Flag as suspicious, require re-authentication
"error" Invalid metadata, missing fields, or comparison issue Check fingerprint structure and try again

üéØ Strictness Levels (Optional Parameter)

  • low: Forgiving. Accepts small changes. Best for mobile/public WiFi users.
  • medium (Default): Standard security. Flags meaningful anomalies.
  • high: Strict. Flags ANY environment deviations. Best for admin panels.

8. Capturing Browser Fingerprints

Basic Usage

import { captureFingerprint } from 'sessionhawk';

async function getFingerprint() {
  const fingerprint = await captureFingerprint();
  console.log(fingerprint);
  return fingerprint;
}

Advanced Options

Exclude heavy computations if performance is a concern:

// Exclude WebGL
const fp = await captureFingerprint({ exclude: 'webgl' });

// Exclude Canvas
const fp = await captureFingerprint({ exclude: 'canvas' });

What is Captured?

User Agent, Screen resolution, Timezone, Plugins, Canvas/WebGL signatures, Audio context, Hardware concurrency, Device memory, Network info.

9. Complete Website Integration

This is a production-ready implementation showing how to register fingerprints at login and validate sessions on every page load.

Part 1: Capture Fingerprint at Login

import { captureFingerprint } from 'sessionhawk';

async function handleUserLogin(username, password) {
  // 1. Authenticate user normally
  const authSuccess = await authenticateUser(username, password);
  if (!authSuccess) return { error: 'Invalid credentials' };
  
  // 2. Capture device fingerprint
  const fingerprint = await captureFingerprint();
  
  // 3. Store fingerprint in SessionHawk
  try {
    await fetch(`http://your-server:8080/groups/main/users/${username}/fp`, {
      method: 'POST',
      headers: {
        'X-API-Key': process.env.VITE_SESSIONHAWK_API_KEY,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ fingerprint })
    });
  } catch (error) {
    console.error('SessionHawk registration error:', error);
  }
  
  return { success: true };
}

Part 2: Validate Session on Every Page Load

Place this in your app's main layout or middleware.

async function validateCurrentSession(strictnessLevel = 'medium') {
  const username = getCurrentUser(); 
  if (!username) return;

  const fingerprint = await captureFingerprint();

  try {
    const response = await fetch('http://your-server:8080/compare', {
      method: 'POST',
      headers: {
        'X-API-Key': process.env.VITE_SESSIONHAWK_API_KEY,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        group: 'main',
        user: username,
        fingerprint,
        strict: strictnessLevel
      })
    });
    
    const result = await response.json();
    
    if (!result.verdict) {
      alert('‚ö†Ô∏è Security Alert: Suspicious activity detected.');
      await logoutUser();
      window.location.href = '/login';
    }
  } catch (error) {
    console.error('Validation error:', error);
  }
}

Part 3: Advanced Patterns

Route-Based Strictness:

const ROUTE_STRICTNESS = {
  '/': 'low',
  '/dashboard': 'medium',
  '/admin': 'high',
  '/payment': 'high',
};

const strictness = ROUTE_STRICTNESS[window.location.pathname] || 'medium';
validateCurrentSession(strictness);

10. Production Best Practices

üîí Security

  • Use HTTPS (reverse proxy with nginx or Traefik).
  • Never expose admin API keys to frontend code.
  • Rotate API keys quarterly.
  • Enable rate limiting on your reverse proxy.

üèóÔ∏è Infrastructure & Backup

  • Run SessionHawk on a dedicated instance/container.
  • Backup Strategy: Run a daily cron job to tarball the data/ directory.
  • Monitor disk usage of the ./data/ directory.

Sample Backup Script

#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
tar -czf backups/sessionhawk_data_$DATE.tar.gz data/
# Delete backups older than 30 days
find backups/ -name "*.tar.gz" -mtime +30 -delete

12. Troubleshooting

Container won't start

Check if port 8080 is in use (netstat -tuln | grep 8080). If so, map a different port (e.g., -p 9090:8080).

Data lost after update

You likely forgot the volume mount flag. Always use -v $PWD/data:/app/data. If you didn't have a backup, the data is lost.

API returns 401 Unauthorized

Check the API key in the docker logs. Ensure the header is exactly X-API-Key: sk_abc... with no extra spaces.

High false positive rate

Try lowering the strictness level to "low". Consider excluding unstable signals like WebGL: captureFingerprint({ exclude: 'webgl' }).
